**Core Idea:**

Software engineering principles were designed for human developers, but software engineering tasks are increasingly performed by AI agents.

This raises the question of whether traditional project structures remain effective under agent-driven workflows.

**Problem Statement:**

It is unclear how different project structures affect the ability of AI agents to navigate, understand, and modify code when performing typical software engineering tasks.

**RQ1**

How does project structure affect agent performance on software engineering tasks?

**RQ2**

Does the effect of project structure differ across task types (e.g., bug fixing, feature addition, refactoring)?

**RQ3**

Is there a point at which increased structural complexity begins to degrade agent performance?

**Hypothesis:**

Project structure influences agent performance, and increased structural complexity may introduce additional navigation overhead that negatively impacts task success.

**Contribution**

This work provides an empirical evaluation of how different project structures impact AI agent performance across common software engineering tasks like bug fixing, feature addition, refactoring.
